Design a modern web application prototype called “RoomSync: Smart Dormitory Operations Management System.”

The interface should be clean, minimal, and professional, suitable for a dormitory and property management platform. Use a minimal color palette: white background, light gray sections, dark gray text, and a soft muted blue or teal accent color. The design should feel similar to a combination of Airbnb and a SaaS dashboard.

Create a responsive dashboard layout with:

Top navigation bar (RoomSync logo on the left, notifications icon, profile avatar, and logout on the right)

Left sidebar navigation

Main content workspace

The system has three user roles: Admin, Dormitory Owner, and Student/Tenant. Design only the core modules and most important screens needed for the prototype.

Screen 1: Login Page

Create a simple centered login interface.

Elements:

RoomSync logo

Email or username field

Password field

Login button

Register link

Forgot password link

Role selection (Admin, Dormitory Owner, Student/Tenant)

Use a minimal card-style login panel.

Screen 2: Admin Dashboard

Sidebar menu:

Dashboard

Manage Dormitory Owners

Dormitory Listings

System Reports

Announcements

Settings

Dashboard widgets:

Total Dormitories

Total Dormitory Owners

Total Tenants

Available Rooms

Occupied Rooms

Below the widgets create a table showing dormitory listings with columns:
Dorm Name, Owner Name, Location, Status, Action buttons.

Screen 3: Dormitory Owner Dashboard

Sidebar menu:

Dashboard

My Dormitories

Tenant Requests

Maintenance Requests

Payments

Notifications

Profile

Dashboard cards showing:

Total Rooms

Available Rooms

Occupied Rooms

Pending Tenant Requests

Maintenance Requests

Use card widgets and simple charts.

Screen 4: Add Dormitory Page (Core Feature)

Create a form where a dormitory owner can register a dorm.

Form fields:

Dormitory Name

Address

Description

Monthly Price

Room Capacity

Amenities (checkbox list such as WiFi, Aircon, Study Area, Laundry)

Upload dormitory images

Upload 360-degree room image

Include a large interactive map component where the owner can drop a location pin to mark the dormitory location. The map should resemble a Google Maps style interface.

Buttons:
Save Dormitory and Cancel.

Screen 5: Student/Tenant Dashboard

Sidebar menu:

Dashboard

Find Dormitory

Dorm Matching

Roommate Matching

My Requests

Maintenance Requests

Payments

Profile

Dashboard content:

Recommended dormitories

Notifications

Pending dorm requests

Screen 6: Map-Based Dormitory Finder

Create a main interface with a large map view.

Features:

Detect the student’s location

Show dormitories within a 150 meter radius

Dormitories appear as map pins

When a pin is clicked, show a small information card displaying:
Dorm name, monthly price, amenities, distance, and a View Details button.

Screen 7: Dormitory Details Page

Display detailed dormitory information including:

Dormitory name

Monthly rent price

Room capacity

Amenities list

Photo gallery

360-degree room viewer

Location map

Include buttons:
Request to Rent and Schedule Walk-in Visit.

Screen 8: Dorm Matching Page

Create a smart dormitory search interface.

Filters:

Budget range

Distance from location

Amenities

Room capacity

Display results as modern dormitory cards with images and quick details.

Screen 9: Roommate Matching Page

Create a form where students enter roommate preferences.

Fields:

Gender

Course

Year Level

Study habits

Sleep schedule

Cleanliness preference

Below the form show recommended roommate profile cards.

Screen 10: Tenant Request Management

Owner page showing tenant requests.

Display requests in card or table layout with:
Student name, requested dormitory, preferred move-in date, and budget.

Buttons:
Approve, Reject, Message.

Screen 11: Maintenance Request System

Tenant form fields:
Room number, issue type, description, upload photo.

Owner view should show maintenance requests in status columns:
Pending, In Progress, Resolved.

Screen 12: Payment Reminder Page

Create a payment tracking interface showing:
Tenant name, room number, monthly rent, due date, payment status (Paid or Unpaid).

Highlight overdue payments with subtle alert styling.

Design the interface using clean spacing, rounded cards, simple icons, minimal color usage, and a professional SaaS dashboard style appropriate for a dormitory operations management platform.
